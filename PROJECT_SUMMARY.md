## 🗄️ 数据库设计亮点

### 核心表结构（24个表）
1. **玩家系统** (6个表)
   - `players` - 玩家基本信息
   - `player_assets` - 玩家资产管理
   - `player_levels` - 等级系统
   - `player_unlocks` - 功能解锁
   - `player_achievements` - 成就系统
   - `player_settings` - 个人设置

2. **Trade模块** (6个表)
   - `commodities` - 商品信息
   - `price_history` - 价格历史
   - `trade_orders` - 交易订单
   - `auto_trading_rules` - 自动交易规则
   - `market_trends` - 市场趋势
   - `trading_fees` - 交易手续费

3. **Venture模块** (7个表)
   - `adventure_teams` - 冒险队伍
   - `adventure_projects` - 冒险项目
   - `investments` - 投资记录
   - `adventure_results` - 冒险结果
   - `team_equipment` - 队伍装备
   - `project_requirements` - 项目需求
   - `investment_returns` - 投资收益

4. **联动系统** (5个表)
   - `market_events` - 市场事件
   - `adventure_market_impacts` - 冒险对市场的影响
   - `market_adventure_impacts` - 市场对冒险的影响
   - `ecosystem_balance` - 生态平衡
   - `economic_cycles` - 经济周期

## 🎮 游戏循环实现

### Trade循环
1. **情报获取** → 查询市场数据和趋势
2. **市场判断** → 分析价格变化和波动
3. **买入卖出** → 执行交易订单
4. **等待波动** → 监控价格变化
5. **收益结算** → 计算盈亏和重新投资

### Venture循环
1. **观察项目** → 浏览冒险项目列表
2. **风险评估** → 分析项目风险和收益
3. **投资决策** → 选择合适的投资额度
4. **等待结果** → 监控冒险进度
5. **收益分配** → 获得投资回报

### 联动效果
- 冒险成功 → 资源增加 → 价格下跌
- 价格暴涨 → 吸引冒险队伍 → 投资热潮
- 过度开采 → 生态破坏 → 市场转向
